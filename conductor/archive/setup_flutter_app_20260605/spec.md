# Specification: Setup initial Flutter app structure

## Overview
Initialize the foundational Flutter application for FitQuest, establishing a Feature-first folder architecture, and set up the core state management solution. This provides the shell and reactive foundation for future feature tracks.

## Functional Requirements
- Create a new Flutter project using Dart.
- Implement a Feature-first directory structure (e.g., `lib/features/`, `lib/core/`).
- Initialize Riverpod for state management.
- Ensure the app builds and runs on the target platforms (Android/iOS).

## Non-Functional Requirements
- Adhere to the established Dart Style Guide for the project.
- Clean out default boilerplate code generated by Flutter.

## Out of Scope
- Initializing the local database (Hive) at this stage.
- Implementing any specific UI or gameplay mechanics beyond a basic shell.
